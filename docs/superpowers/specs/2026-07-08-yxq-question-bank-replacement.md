# yxq 英语题库替换规格

## 背景

用户希望把已经制作好的英语刷题网站题库，替换为 `D:\学习\大二下\英语阅读\yxq英语题库.docx` 中的题目。

当前网站已经支持从结构化 JSON 题库加载题目，练习模式、背题模式、收藏、错题和本地编辑都围绕 `public/questions.json` 工作。本次目标是替换内置题库数据，不重做网站功能。

## 目标

- 从 `yxq英语题库.docx` 提取题目，替换网站内置题库。
- 保留现有题库 JSON schema、稳定 ID、数据校验、去重和发布流程。
- 网站启动后练习模式和背题模式默认展示新题库。
- 不修改原始 DOCX。
- 不把当前工作区中与本任务无关的未提交改动纳入提交。

## 非目标

- 不合并旧题库。
- 不保留旧 120 题作为内置题库。
- 不新增云同步、账号、后台或数据库。
- 不改变练习模式、背题模式、收藏、错题、本地编辑等交互。
- 不新增依赖，除非现有环境完全无法读取 DOCX，且必须先回报产品部。

## 数据规则

源文档已初步识别为：

- 标题：`yxq英语题库`
- 小标题：`Vocabulary 题目整理（共 60 题）`
- 每题结构：题号、英文题干、A-D 四个选项、`答案：X`

发布题目规则：

- 题型：全部作为 `single_choice`。
- 选项：保留 A、B、C、D 的完整文本。
- 答案：来自 `答案：A/B/C/D`；不得自动猜测缺失答案。
- 来源：每题 source 写为 `yxq英语题库.docx#题号`。
- 分类：如果没有更明确分类，统一为 `other`。
- 校对状态：完整题目设为 `verified`；不完整或答案非法设为 `needs_review`，不得发布到 `public/questions.json`。
- OCR 相关字段：本任务不是 OCR 提取，`ocrConfidence` 使用 `null`。
- ID：继续用题干和选项指纹生成稳定 ID，避免同一道题重新导入时 ID 乱变。

## 验收标准

1. `data/questions.raw.json` 中识别到 60 条来自 `yxq英语题库.docx` 的原始题目记录。
2. 每条题目包含英文题干、A-D 四个选项、一个合法答案、source、category、reviewStatus、fingerprint。
3. `public/questions.json` 只包含校验通过且可发布的题目；若源文档 60 题都完整，应发布 60 题。
4. 旧内置 120 题不再出现在 `public/questions.json`。
5. `data/conversion-report.json` 反映本次新题库的识别数量、发布数量、去重数量、待校对数量和冲突数量。
6. 练习模式和背题模式都能加载新题库，分类筛选不报错。
7. 保留用户本地数据兼容策略：由于题库 ID 已变化，旧题库相关进度可以自然不匹配，但网站不能白屏或崩溃。
8. 原始文件 `D:\学习\大二下\英语阅读\yxq英语题库.docx` 不被修改。
9. 提交范围只包含本任务必要文件；不得提交 `.claude/`、`tsconfig.json` 或其他无关未提交源码改动。
10. 产品验收通过后，执行方提交并推送到现有 GitHub 远端。

## 验证要求

- 数据提取脚本运行成功。
- `npm run convert` 或等价转换命令运行成功。
- `npm run validate:data` 通过。
- 相关单元测试通过，至少覆盖新 DOCX 格式解析或生成结果数量。
- `npm test` 通过。
- `npm run build` 通过。
- 如浏览器自动化可用，打开本地网站确认首页题目总数为新题库数量；不可用时说明限制，并用 HTTP/静态数据检查替代。
