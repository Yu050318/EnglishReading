# Windows 一键启动脚本设计

## 目标

在项目根目录提供可双击运行的 `启动网站.bat`，让用户无需手动打开终端或输入命令即可启动刷题网站，并允许同一 Wi-Fi 下的手机访问。

## 启动流程

1. 脚本始终以自身所在目录作为项目目录，不依赖用户从哪个目录双击。
2. 检查 `node` 与 `npm` 是否可用；任一缺失时显示中文提示，说明需要安装 Node.js，然后暂停退出。
3. 检查 `package.json`；缺失时提示“启动脚本不在正确的项目目录”，暂停退出。
4. 仅当 `node_modules` 不存在时运行 `npm install`；安装失败时显示错误并暂停退出。
5. 使用固定地址 `0.0.0.0`、固定端口 `5173` 和严格端口模式启动 Vite。
6. 启动后自动打开电脑默认浏览器 `http://localhost:5173`。
7. 命令窗口保持打开并显示 Vite 输出；用户关闭窗口或按 `Ctrl+C` 即停止网站。

## 手机访问

- Vite 必须监听 `0.0.0.0`，以便局域网设备访问。
- 脚本启动时显示提示：手机与电脑连接同一 Wi-Fi 后，使用 Vite 输出中的 `Network` 地址访问。
- Windows 防火墙首次询问时，用户需允许 Node.js 使用专用网络。

## 端口与重复启动

- 固定使用 `5173`，并启用严格端口模式。
- 如果端口已被占用，Vite 必须直接报错，不自动切换到其他端口，避免浏览器打开错误地址。
- 脚本在 Vite 退出后显示“网站已停止或启动失败”，并暂停窗口，便于用户阅读错误。

## 安全与范围

- 脚本不得修改系统 PATH、防火墙、代理、Git 配置或项目源码。
- 不自动提升管理员权限。
- 不在每次启动时重复安装依赖。
- 不引入新的 npm 包。

## 验收标准

1. 从资源管理器双击 `启动网站.bat` 可以正确定位项目目录。
2. Node.js/npm 缺失、`package.json` 缺失和依赖安装失败均有明确提示并以失败状态结束。
3. `node_modules` 已存在时不会运行 `npm install`。
4. 启动命令使用 `--host 0.0.0.0 --port 5173 --strictPort`。
5. 自动打开 `http://localhost:5173`，且窗口保持显示服务器日志。
6. 端口被占用时不会改用其他端口。
7. README 增加双击启动、手机访问和停止网站说明。
8. 启动脚本不得包含机器专属绝对路径。
