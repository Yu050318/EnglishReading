# 多科目与人力资源题库规格

## 背景

当前英语刷题网站已经有一个内置英语题库，用户希望在不改变原有功能和已有题库的基础上，新增 `D:\学习\大二下\英语阅读\人力期中考核总复习_修正版答案解析.docx` 作为第二个题库，并把两个题库分成两个科目：

- 原有题库：英语
- 新增题库：人力资源

初步检查新 DOCX 内容包含：

- 单项选择题
- 不定项选择题
- 计算题
- 案例分析题

## 产品目标

- 网站从“单一英语题库”升级为“多科目刷题网站”。
- 原有英语题库、英语学习记录、错题、收藏、题库管理能力继续可用。
- 新增“人力资源”科目，使用新 DOCX 作为题库来源。
- 英语和人力资源在入口、题库、练习/背题队列、统计、错题、收藏上彼此分开。
- 保留 Vercel 静态部署与本地 LocalStorage / 当前云同步能力的兼容。

## 非目标

- 不删除、不替换英语题库。
- 不把人力资源题库混入英语科目。
- 不为计算题/案例题做自动主观评分。
- 不新增账号体系、排行榜、后台管理或数据库表结构，除非当前云同步实现要求 subject 字段进入同步 payload。
- 不修改源 DOCX。

## 科目设计

新增科目模型：

```ts
type SubjectId = 'english' | 'human_resources';
```

显示文案：

- `english` → `英语`
- `human_resources` → `人力资源`

题目记录新增或等效支持：

```ts
subject: SubjectId
```

规则：

- 所有现有英语内置题目补齐 `subject: "english"`。
- 新 DOCX 提取的人力资源题目全部为 `subject: "human_resources"`。
- 若导入旧 JSON 没有 subject，为兼容旧数据，默认视为 `english`，但重新保存/导出时应补齐 subject。

## 题型设计

现有题型继续支持：

- `single_choice`
- `true_false`

新增题型：

- `multiple_choice`：不定项选择题，支持多个正确答案，只有完全一致才判正确。
- `short_answer`：计算题、案例分析题等主观题，只用于背题/题库浏览；练习模式中可显示“查看参考答案”，不纳入自动正确率，或默认不进入自动判分练习队列。

人力资源 DOCX 提取规则：

- “一、单项选择题”：生成 `single_choice`。
- “二、不定项选择题”：生成 `multiple_choice`，答案如 `BD` 拆成 `["B","D"]`。
- “三、计算题”：生成 `short_answer`，保留题干、参考答案、解析。
- “四、案例分析题”：生成 `short_answer`，保留题干、参考答案、解析。

字段规则：

- 选择题保留 A-D 选项。
- `答案：` 后内容写入 `answer`；多选拆分为多个字母。
- `解析：` 后内容写入 `explanation`。
- 主观题参考答案可放入 `answerText` 或等效字段；不得塞进 A-D 选项。
- source 使用 `人力期中考核总复习_修正版答案解析.docx#题号或章节题号`。
- 分类 category 可先统一为 `human_resources_review` 或 `other`，但页面显示应优先展示科目“人力资源”，不让用户误以为是英语词汇题库。

## 交互设计

### 首页

首页增加科目选择区，至少显示两个科目卡片：

- 英语
  - 显示英语题库数量、练习入口、背题入口。
- 人力资源
  - 显示人力资源题库数量、练习入口、背题入口。

用户选择科目后，后续页面只使用该科目的题目。

### 导航与页面

保留原有导航结构，但当前科目需要清晰可见，例如：

- 当前科目：英语 / 人力资源
- 可切换回首页或科目选择。

练习、背题、错题、收藏、题库管理均按当前科目过滤。

### 练习模式

- 英语保持现有体验。
- 人力资源：
  - 单选题可正常判分。
  - 多选题支持多选提交，完全匹配答案才算正确。
  - 主观题默认不进入自动判分练习队列；如果出现在练习中，应只提供“查看参考答案/下一题”，不计入正确率。

### 背题模式

- 支持英语和人力资源。
- 人力资源主观题必须能展示参考答案和解析。
- 跳题功能继续按当前科目和筛选后的队列编号。

### 错题、收藏、进度

- 同一个题目 ID 必须稳定，并包含科目前缀或能避免跨科目冲突。
- 错题、收藏、进度按 subject 隔离显示。
- 旧英语学习记录尽量不丢失；旧记录没有 subject 时按英语处理。

## 数据文件建议

可以采用以下任一实现，只要最终行为满足验收：

方案 A（推荐）：一个 `public/questions.json` 包含全部科目题目，每题带 subject。

- 优点：改动较少，现有加载逻辑继续请求一个文件。
- 要求：所有筛选、统计、导入导出都理解 subject。

方案 B：分文件，例如 `public/subjects/english.json` 与 `public/subjects/human-resources.json`。

- 优点：科目边界清晰。
- 缺点：现有加载和导入导出改动更大。

推荐采用方案 A，除非执行时发现当前代码结构更适合方案 B。

## 验收标准

1. 网站至少有两个科目入口：英语、人力资源。
2. 英语题库仍存在，题目数量和分类不被人力资源文档覆盖。
3. 人力资源题库从 `人力期中考核总复习_修正版答案解析.docx` 提取，包含单选、多选、计算题、案例分析题。
4. 英语页面只显示英语题，人力资源页面只显示人力资源题。
5. 人力资源单选题可判分。
6. 人力资源多选题可选择多个选项，并按完全一致判分。
7. 人力资源计算题/案例题在背题和题库管理中能看到参考答案与解析，不被错误当成普通单选题。
8. 错题、收藏、进度按科目隔离；英语旧记录不因新增人力资源题库失效或污染。
9. 题库导入/导出仍可用；旧 JSON 没有 subject 时按英语兼容。
10. 生产构建通过，Vercel 静态托管不受影响。
11. 不提交 `.claude/`、`.env*`、本地 Vercel 文件或其他任务外 dirty 文件。

## 验证要求

- 新增/更新数据 schema 测试。
- 新增人力资源 DOCX 提取测试，至少覆盖：
  - 单选题答案 B。
  - 多选题答案 BD。
  - 计算题参考答案。
  - 案例题参考答案。
- 新增多选判分测试。
- 新增 subject 过滤/隔离测试。
- 运行：
  - `npm.cmd run validate:data`
  - `npm.cmd test`
  - `npm.cmd run build`
  - `git diff --check`
- 可用时做浏览器验证：
  - 首页可切换英语/人力资源。
  - 人力资源背题能看到计算题或案例题参考答案。
  - 英语题库仍可练习。
