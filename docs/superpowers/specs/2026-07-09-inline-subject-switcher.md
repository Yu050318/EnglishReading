# 同页科目切换菜单规格

## 背景

当前多科目版本中，页面显示：

> 当前科目：英语 · 切换科目

点击“切换科目”会回到首页。用户在手机截图中反馈，希望不要返回首页，而是在当前页面直接显示科目菜单，并在当前功能页切换到对应科目的内容。

示例诉求：

- 当前在“错题本”页面时，切换到“人力资源”后仍在“错题本”，只显示人力资源错题。
- 当前在“背题”页面时，切换到“英语/人力资源”后仍在“背题”页面。

## 目标

- 把“切换科目”链接替换为同页科目菜单。
- 用户可在当前页面直接切换“英语 / 人力资源”。
- 切换后保持当前页面/功能不变，只切换 subject。
- 移动端可用，不产生横向溢出。
- 不改变题库、判分、云同步、Vercel/Supabase 配置。

## 非目标

- 不重新设计整体导航。
- 不删除首页科目入口；首页入口可保留。
- 不改变科目数据、错题/收藏/进度规则。
- 不新增新的科目管理后台。

## 交互设计

在练习、背题、错题、收藏、题库管理等功能页，将当前科目区域改为：

```text
当前科目：
[ 英语 ] [ 人力资源 ]
```

或等效的移动端友好菜单/分段控件。

规则：

- 当前科目按钮高亮。
- 点击另一个科目后，仍停留在当前路由。
- URL 的 `subject` 参数更新为目标科目。
- 页面内容立即切换到目标科目。
- 对需要初始化队列的页面：
  - 背题：切换科目后从该科目第 1 题开始。
  - 练习设置页：切换科目后仍在练习设置页。
  - 如果正在答题中，切换科目应回到该科目的练习设置状态，避免把两个科目的答题队列混在一起。
  - 错题/收藏/题库：直接展示目标科目的列表。

## 文案

- 标题：`当前科目：`
- 按钮：`英语`、`人力资源`
- 不再显示“切换科目”文字链接。

## 验收标准

1. 功能页不再出现“切换科目”链接。
2. 功能页显示“英语 / 人力资源”科目菜单。
3. 在错题本页面切换科目后，仍在错题本页面。
4. 在背题页面切换科目后，仍在背题页面，并显示目标科目的题目。
5. 在练习页面切换科目后，不混用旧科目的练习队列。
6. 当前科目有明确高亮或选中状态。
7. 375px 移动端无横向溢出。
8. 云同步状态条、顶部导航、题库数据不受影响。
9. 完成后提交、推送并部署到 Vercel production。

## 验证要求

- 单元或组件/静态测试覆盖 subject 切换 URL 生成逻辑。
- `npm.cmd run validate:data`
- `npm.cmd test`
- `npm.cmd run build`
- `git diff --check`
- 线上或本地浏览器验证：
  - `/#/mistakes?subject=english` 切到人力资源后仍是 `#/mistakes?...`
  - `/#/memorize?subject=english` 切到人力资源后仍是 `#/memorize?...`
  - 页面不再出现“切换科目”
  - 375px 无横向溢出
