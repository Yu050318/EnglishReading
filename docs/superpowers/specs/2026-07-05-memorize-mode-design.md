# 背题模式设计

## 目标

在现有英语阅读刷题网站中增加独立的“背题”页面，让用户无需作答即可同时查看题目、全部选项和正确答案。

## 交互

- 顶部导航和首页增加“背题”入口，路由为 `/memorize`。
- 每次只显示一道题，并显示当前位置，例如 `12 / 120`。
- 保留全部选项；正确选项使用绿色样式高亮。
- 选项下方明确显示正确答案字母与对应文本；存在解析时同时显示解析。
- 提供“上一题”和“下一题”按钮，首题禁用上一题，末题禁用下一题。
- 支持键盘左右方向键切换题目。
- 支持按现有题目分类筛选；切换分类后回到该分类第一题。
- 背题浏览不写入练习进度、正确率或错题本；收藏功能继续可用。

## 架构

- 新增纯函数负责分类后的背题队列与安全索引，放在独立的背题功能模块中，便于单元测试。
- 新增 `Memorize` React 页面组件，复用现有 `Question`、分类标签、收藏状态和纸张批注风格。
- 不修改题库 JSON 格式，也不新增 LocalStorage 字段。

## 异常与空状态

- 题库尚未加载时显示“题库加载中”。
- 分类下没有题目时显示空状态，并允许切换回全部分类。
- 页面索引始终限制在有效范围内，避免筛选后访问不存在的题目。

## 验证

- 单元测试覆盖分类队列、首尾索引和空队列。
- 回归运行完整 Vitest、题库校验和 Vite 生产构建。
- 检查桌面与 375px 手机布局，确保按钮和选项无横向溢出。
